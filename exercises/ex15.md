# Ex15 — Resposta final

## Contexto

O usuário não quer um JSON cru. Ele quer uma resposta útil, com evidências e próximo passo.

Este node tambem nao deve depender da implementacao do servidor. Ele recebe o `context` ja preenchido por tools/resources/prompts MCP.

## Arquivo permitido

```text
stack_sentinel/agent/nodes.py
```

## Contrato

Implemente:

```python
final_answer_node(state: AgentState) -> AgentState
```

Regras:

- para ticket, inclua ID, resumo, severidade, serviço, status e próximo passo;
- para build, inclua ID, status, serviço, etapa falha e próximo passo;
- para docs, use o contexto de resource/prompt;
- para `unknown` ou erro, retorne fallback claro;
- a resposta deve conter seções ou marcadores equivalentes a `Resumo`, `Evidências` e `Próximo passo`;
- salve a resposta em `state["final_answer"]`.

## Validação

```bash
python run.py test ex15
```

## Desafios opcionais

- Faça a resposta final citar explicitamente quais capacidades MCP foram usadas, como `fetch_ticket_context`, `fetch_build_status`, resource de documentação ou prompt de triagem. Isso torna a resposta mais auditável e ajuda o usuário a confiar no caminho percorrido.
- Adicione tratamento para contexto parcial, gerando uma resposta útil mesmo quando faltarem alguns campos esperados. A resposta deve diferenciar claramente evidência disponível de informação ausente, sem inventar valores.
