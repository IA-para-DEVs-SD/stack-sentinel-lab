# Ex07 — Criar prompt MCP

## Contexto

Prompts MCP padronizam como clientes e agentes devem agir dentro de um domínio. Aqui, o prompt orienta uma triagem de incidente e deve ser exposto no `FastMCP` com `@mcp.prompt()`.

## Arquivo permitido

```text
stack_sentinel/mcp_server/prompts.py
stack_sentinel/mcp_server/fastmcp_server.py
```

## Contrato

Implemente:

```python
incident_triage_prompt(user_question: str, available_context: str) -> str
```

O texto retornado deve orientar o agente a:

- resumir o problema;
- citar severidade quando houver evidência;
- sugerir próximo passo;
- não inventar dados ausentes;
- usar a pergunta do usuário;
- usar o contexto disponível.
- registrar `incident_triage_prompt` no servidor `FastMCP`.

## Validação

```bash
python run.py test ex07
```

## Desafios opcionais

- Crie um segundo prompt para análise de build quebrado usando `BUILD_FAILURE_ANALYSIS_PROMPT`. Ele deve orientar o agente a usar status, etapa falha e trecho de log para formular hipóteses, mas também deixar claro quando não houver evidência suficiente.
- Reescreva o prompt de triagem para exigir uma resposta com estrutura fixa, como `Resumo`, `Evidências`, `Lacunas` e `Próximo passo`. O foco é perceber como prompts MCP podem funcionar como contrato de comportamento, não apenas como texto livre.
