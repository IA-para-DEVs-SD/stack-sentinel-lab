# Ex08 — Criar grafo mínimo

## Contexto

Antes de integrar MCP, crie um grafo mínimo para entender a mecânica de execução: um state entra, passa por um node e sai atualizado.

Na arquitetura atualizada, este grafo ainda roda em memoria. A integracao com o servidor `FastMCP` aparece mais adiante por meio do cliente MCP.

## Arquivo permitido

```text
stack_sentinel/agent/graph.py
```

## Contrato

Implemente:

```python
compile_minimal_graph() -> SimpleGraph
```

Regras:

- retorne uma instância de `SimpleGraph`;
- adicione pelo menos um node usando `graph.add_node`;
- ao executar `graph.run({"user_input": "ping"})`, o resultado deve conter `final_answer == "ping"`;
- o node deve preservar `user_input`.

## Validação

```bash
python run.py test ex08
```

## Desafios opcionais

- Adicione suporte a nomes de nodes no `SimpleGraph`, permitindo registrar algo como `graph.add_node(echo_node, name="echo")`. Isso deve facilitar debug sem mudar o contrato básico de execução dos nodes.
- Faça o grafo acumular no state um histórico simples dos nodes executados. O histórico deve ajudar o aluno a visualizar a ordem do fluxo, mas não deve impedir que os testes existentes continuem usando states mínimos.
