# Ex08 — Criar grafo mínimo com LangGraph

## Contexto

Antes de integrar MCP, crie um grafo mínimo usando LangGraph de verdade. A mecânica central é: um `state` entra no grafo, passa por um node e sai atualizado.

Na arquitetura atualizada, este grafo ainda roda em memoria. A integracao com o servidor `FastMCP` aparece mais adiante por meio do cliente MCP.

## Arquivo permitido

```text
stack_sentinel/agent/graph.py
```

## Contrato

Implemente:

```python
compile_minimal_graph()
```

Regras:

- use `StateGraph` de `langgraph.graph`;
- use `AgentState` como schema do state;
- adicione pelo menos um node usando `graph.add_node`;
- conecte `START -> node -> END`;
- retorne o grafo compilado com `graph.compile()`;
- ao executar `graph.invoke({"user_input": "ping"})`, o resultado deve conter `final_answer == "ping"`;
- o node deve preservar `user_input`.

## Validação

```bash
python run.py test ex08
```

## Desafios opcionais

- Dê nomes claros aos nodes e inspecione `graph.get_graph()` depois da compilação. O objetivo é enxergar a topologia gerada pelo LangGraph, não apenas o resultado final.
- Faça o node acumular no state um histórico simples dos passos executados. O histórico deve ajudar o aluno a visualizar a ordem do fluxo, mas não deve impedir que os testes existentes continuem usando states mínimos.
