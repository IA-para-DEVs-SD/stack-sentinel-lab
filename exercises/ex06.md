# Ex06 — Criar resource MCP

## Contexto

Nem tudo é ação. Algumas informações são documentação, política ou contexto. Resources representam dados legíveis e agora também devem ser expostos no servidor `FastMCP`.

## Arquivo permitido

```text
stack_sentinel/mcp_server/resources.py
stack_sentinel/mcp_server/fastmcp_server.py
```

## Contrato

Implemente:

```python
read_doc_resource(uri: str, client: Optional[MockServiceClient] = None) -> dict
```

Em sucesso, retorne:

```python
{
    "ok": True,
    "uri": "docs://incident-response",
    "title": "Incident Response Guide",
    "content": "..."
}
```

Em falha, retorne `ok=False` e `error`.

Regras:

- aceite apenas URIs presentes em `RESOURCE_TO_SLUG`;
- use `client.get_doc(slug)`;
- preserve o campo `uri` original na resposta.
- registre pelo menos `docs://incident-response` no `FastMCP` usando o handler de `resources.py`.

## Validação

```bash
python run.py test ex06
```

## Desafios opcionais

- Registre automaticamente todos os resources retornados por `list_doc_resources()` no `FastMCP`. O desafio está em capturar corretamente cada `uri` dentro do loop, evitando o bug clássico em que todas as funções acabam apontando para o último item.
- Implemente um helper que converta a resposta estruturada de qualquer resource em texto amigável. Ele deve preservar título e conteúdo, mas também tratar erros de forma legível para clientes MCP que esperam texto em vez de dicionário.
