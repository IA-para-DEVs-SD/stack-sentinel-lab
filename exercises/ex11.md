# Ex11 — Roteamento condicional

## Contexto

Agora o fluxo deixa de ser linear. A intenção classificada define qual node deve executar em seguida. Em LangGraph, essa função pode ser usada como função de roteamento em `add_conditional_edges`.

As rotas representam capacidades do agente, nao detalhes do transporte MCP. Por isso elas continuam iguais mesmo com `FastMCP`.

## Arquivo permitido

```text
stack_sentinel/agent/graph.py
```

## Contrato

Implemente:

```python
route_by_intent(state: AgentState) -> str
```

Mapeamento obrigatório:

- `ticket` -> `fetch_ticket`;
- `build` -> `fetch_build`;
- `docs` -> `fetch_docs`;
- qualquer outro valor -> `fallback`.

Esses nomes devem corresponder aos nomes dos nodes que serão usados no grafo final.

## Validação

```bash
python run.py test ex11
```

## Desafios opcionais

- Permita rotas compostas quando o state indicar mais de uma necessidade, por exemplo consultar ticket e depois build quando ambos os IDs estiverem presentes. Pense em como representar essa rota sem quebrar o mapeamento simples exigido no exercício obrigatório.
- Crie uma forma de retornar ou registrar uma justificativa curta para a decisão de roteamento. A justificativa deve ser útil para auditoria, por exemplo explicando qual `intent` ou quais campos do state levaram à rota escolhida.
