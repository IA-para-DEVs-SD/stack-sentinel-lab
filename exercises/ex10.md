# Ex10 — Criar node de classificação

## Contexto

O agente precisa transformar uma pergunta em intenção de rota. Os testes usam `FakeLLMClient` para que a classificação seja previsível.

Este é um node de LangGraph: ele recebe `AgentState` e devolve uma atualização de state. O node nao deve depender diretamente de `FastMCP`; ele apenas decide a rota antes de qualquer chamada MCP.

## Arquivo permitido

```text
stack_sentinel/agent/nodes.py
```

## Contrato

Implemente:

```python
classify_intent_node(state: AgentState, llm: LLMClient) -> AgentState
```

Regras:

- chame `llm.classify_intent(state["user_input"])`;
- aceite apenas `ticket`, `build`, `docs` ou `unknown`;
- se a LLM retornar algo fora da lista, use `unknown`;
- retorne um novo state, sem mutar o original;
- preserve os demais campos existentes.
- mantenha o retorno compatível com o schema `AgentState` usado no `StateGraph`.

## Validação

```bash
python run.py test ex10
```

## Desafios opcionais

- Faça a classificação também preencher `ticket_id` e `build_id` quando esses identificadores aparecerem no texto do usuário. Isso antecipa trabalho dos nodes seguintes e mostra como um node pode enriquecer o state além de definir a rota.
- Adicione um fallback determinístico por palavras-chave quando a LLM retornar `unknown`. A regra deve ser simples e previsível, por exemplo reconhecer termos como "ticket", "build" ou "documentação", sem substituir a LLM quando ela já retornar uma intenção válida.
