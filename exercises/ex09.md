# Ex09 — Definir AgentState para LangGraph

## Contexto

O state é o contrato que atravessa os nodes do agente no LangGraph. Ele funciona como o schema usado pelo `StateGraph`, então cada node deve receber e devolver atualizações compatíveis com esse formato.

Esse state é independente da implementação do servidor: tanto um cliente didático quanto um cliente conectado a `FastMCP` devem conseguir preencher os mesmos campos.

## Arquivo permitido

```text
stack_sentinel/agent/state.py
```

## Contrato

Implemente:

```python
create_initial_state(user_input: str) -> AgentState
```

O retorno deve conter:

- `user_input` com o texto recebido;
- `intent`;
- `ticket_id`;
- `build_id`;
- `context`;
- `error`;
- `final_answer`.

Todos os campos exceto `user_input` devem começar com `None`.

O tipo `AgentState` deve continuar acessível como um `TypedDict`, para que possa ser passado para `StateGraph(AgentState)`.

## Validação

```bash
python run.py test ex09
```

## Desafios opcionais

- Adicione campos opcionais como `trace_id` e `steps` para pensar em observabilidade do agente. Eles não precisam ser usados pelos testes, mas devem abrir caminho para rastrear uma execução de ponta a ponta.
- Experimente complementar o `TypedDict` com validação externa, como uma função de validação ou modelo Pydantic. O desafio é manter compatibilidade com LangGraph e com os nodes que acessam o state como dicionário.
