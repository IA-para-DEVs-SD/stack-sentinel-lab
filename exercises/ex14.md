# Ex14 — Node que usa resource/prompt

## Contexto

Nem toda resposta precisa chamar uma tool. Para documentação e triagem, o agente pode buscar um resource e um prompt MCP.

No servidor real, esses recursos e prompts foram registrados no `FastMCP` nos Ex06 e Ex07.

## Arquivo permitido

```text
stack_sentinel/agent/nodes.py
```

## Contrato

Implemente:

```python
fetch_docs_node(state: AgentState, mcp_client: MCPClient) -> AgentState
```

Regras:

- leia `INCIDENT_RESPONSE_RESOURCE` com `mcp_client.read_resource`;
- obtenha `INCIDENT_TRIAGE_PROMPT` com `mcp_client.get_prompt`;
- passe `user_question` e `available_context` como argumentos do prompt;
- salve em `context` um dicionário com as chaves `resource` e `prompt`;
- em falha, preencha `error`.

## Validação

```bash
python run.py test ex14
```

## Desafios opcionais

- Escolha dinamicamente o resource mais adequado a partir da pergunta do usuário. Por exemplo, perguntas sobre severidade podem usar a política de severidade, enquanto perguntas sobre falha de build podem usar o playbook de build.
- Combine mais de um resource antes de montar o prompt final. O desafio é juntar conteúdos sem perder a origem de cada documento, para que a resposta final possa explicar de onde vieram as evidências.
