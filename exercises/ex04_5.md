# Ex04.5 — Refatorar o servidor para FastMCP

## Contexto

Ate aqui, o lab usa `SimpleMCPServer`, uma implementacao didatica que guarda tools, resources e prompts em dicionarios Python. Isso facilita testes deterministicos e deixa os contratos visiveis.

Agora vamos virar a chave: os proximos exercicios passam a considerar `FastMCP` como a forma principal de expor capacidades MCP. O `SimpleMCPServer` pode continuar existindo como harness de testes em memoria, mas a arquitetura da aula deve apontar para `FastMCP`.

Instale as dependencias do projeto antes de continuar:

```bash
pip install -r requirements.txt
```

## Arquivo a criar

```text
stack_sentinel/mcp_server/fastmcp_server.py
```

## Contrato

Crie uma funcao:

```python
create_fastmcp_server()
```

Ela deve:

- importar `FastMCP` de `mcp.server.fastmcp`;
- criar um servidor com o nome `stack-sentinel-mcp`;
- registrar a tool `fetch_ticket_context`;
- delegar a execucao para `stack_sentinel.mcp_server.tools.fetch_ticket_context`;
- nao reimplementar a regra da tool dentro do arquivo novo.

## Exemplo de fluxo esperado

```text
FastMCP recebe chamada da tool
  -> handler fetch_ticket_context(ticket_id)
  -> tools.fetch_ticket_context(ticket_id)
  -> MockServiceClient
  -> Mock API FastAPI
```

## Validacao manual

Depois de implementar, rode:

```bash
python -c "from stack_sentinel.mcp_server.fastmcp_server import create_fastmcp_server; print(create_fastmcp_server())"
```

Se o import do pacote `mcp` falhar, confirme que voce rodou `pip install -r requirements.txt`.

## Importante

Nao altere:

- `SimpleMCPServer`;
- `create_mcp_server`;
- `register_ticket_tool`;
- os testes existentes;
- os contratos de `tools.py`.

Os exercicios seguintes devem manter as regras de dominio em `tools.py`, `resources.py` e `prompts.py`, mas a exposicao MCP real passa a ser feita no arquivo `fastmcp_server.py`.

## Desafios opcionais

Se terminar rapido, escolha um ou mais:

- Crie uma função `run_fastmcp_server()` para executar o servidor manualmente. Ela deve chamar `create_fastmcp_server()` e iniciar o servidor, mas o arquivo ainda precisa poder ser importado pelos testes sem subir processo, abrir porta ou bloquear o terminal.
- Proponha uma estratégia pequena para evitar duplicação entre o registro didático em `server.py` e o registro real em `fastmcp_server.py`. A ideia é separar metadados e handlers compartilhados da camada específica de registro, sem esconder demais o que está acontecendo para fins didáticos.
