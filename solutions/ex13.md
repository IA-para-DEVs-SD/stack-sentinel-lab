Substitua a função `fetch_build_node` por:

```python
def fetch_build_node(state: AgentState, mcp_client: MCPClient) -> AgentState:
    build_id = state.get("build_id") or extract_build_id(state.get("user_input", ""))
    if not build_id:
        return update_state(state, build_id=None, context=None, error="build_id not found")

    result = mcp_client.call_tool(BUILD_TOOL_NAME, {"build_id": build_id})
    if not result.get("ok"):
        return update_state(state, build_id=build_id, context=None, error=result.get("error", "build tool failed"))

    return update_state(state, build_id=build_id, context=result, error=None)
```

## Solução alternativa válida

Assim como no ticket, a extração pode vir do state ou do texto. A ordem que passa nos testes é:

```python
build_id = state.get("build_id") or extract_build_id(state.get("user_input", ""))
```

O node não deve chamar a FastAPI diretamente neste exercício. A camada correta aqui é MCP Client → MCP Server FastMCP → Tool.

def classify_build_failure(result: dict) -> str:
    failed_step = result.get("failed_step", "")
    log_excerpt = result.get("log_excerpt", "")
    if "integration" in failed_step:
        return "integration"
    if "timeout" in log_excerpt.lower():
        return "timeout"
    return "unknown"
