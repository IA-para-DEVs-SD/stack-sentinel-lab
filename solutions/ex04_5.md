# Solucao Ex04.5 — Refatorar o servidor para FastMCP

Crie o arquivo:

```text
stack_sentinel/mcp_server/fastmcp_server.py
```

Uma solucao simples:

```python
from stack_sentinel.mcp_server import tools


def create_fastmcp_server():
    from mcp.server.fastmcp import FastMCP

    mcp = FastMCP("stack-sentinel-mcp")

    @mcp.tool()
    def fetch_ticket_context(ticket_id: str) -> dict:
        """Busca contexto normalizado de um ticket."""
        return tools.fetch_ticket_context(ticket_id)

    return mcp
```

## Por que esta solucao e suficiente?

O ponto deste exercicio nao e reescrever o servidor MCP inteiro. E perceber que o handler de dominio continua o mesmo:

```python
tools.fetch_ticket_context(ticket_id)
```

O que muda e a camada de exposicao:

```text
SimpleMCPServer.register_tool(...)
```

vira:

```text
@mcp.tool()
```

## Alternativa valida

Tambem e aceitavel importar a funcao diretamente:

```python
from stack_sentinel.mcp_server.tools import fetch_ticket_context


def create_fastmcp_server():
    from mcp.server.fastmcp import FastMCP

    mcp = FastMCP("stack-sentinel-mcp")
    mcp.tool()(fetch_ticket_context)
    return mcp
```

Essa alternativa e mais curta, mas a primeira costuma ser mais facil de explicar em aula porque deixa explicita a delegacao.

## O que nao fazer

Nao copie a logica de `fetch_ticket_context` para dentro de `fastmcp_server.py`. O servidor MCP deve expor capacidades; a regra da capacidade continua morando em `tools.py`.

## Desafios opcionais — solucao proposta

Para executar o servidor manualmente, uma solucao simples e adicionar:

```python
def run_fastmcp_server() -> None:
    create_fastmcp_server().run()


if __name__ == "__main__":
    run_fastmcp_server()
```

Com isso, o arquivo pode ser executado manualmente com:

```bash
python -m stack_sentinel.mcp_server.fastmcp_server
```

Quando o arquivo for apenas importado, nada sobe automaticamente. Isso e importante para testes e para outros modulos que so precisam chamar `create_fastmcp_server()`.

Para reduzir duplicacao, mantenha nomes e handlers em constantes compartilhadas ou funcoes pequenas como `register_ticket_capability(mcp)`. Evite misturar regra de negocio com decoradores: o handler FastMCP deve continuar chamando `tools.fetch_ticket_context`.
