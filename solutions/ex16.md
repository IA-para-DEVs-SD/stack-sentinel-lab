## Solução alternativa válida

Você pode organizar o roteamento com um dicionário de funções em vez de `if/elif`, desde que o fluxo seja o mesmo.

```python
def run_stack_sentinel_flow(state: AgentState, llm: LLMClient, mcp_client: MCPClient) -> AgentState:
    current = classify_intent_node(state, llm)
    route = route_by_intent(current)

    handlers = {
        "fetch_ticket": lambda s: fetch_ticket_node(s, mcp_client),
        "fetch_build": lambda s: fetch_build_node(s, mcp_client),
        "fetch_docs": lambda s: fetch_docs_node(s, mcp_client),
        "fallback": fallback_node,
    }

    current = handlers.get(route, fallback_node)(current)
    return final_answer_node(current)
```

O importante é manter as quatro etapas: classificar, rotear, executar o node da rota e gerar resposta final.

## Desafios opcionais — solucao proposta

Para acumular steps:

```python
def add_step(state: AgentState, step: str) -> AgentState:
    return {**state, "steps": [*(state.get("steps") or []), step]}
```

Chame `add_step` depois de classificar, rotear, buscar contexto e gerar resposta.

Para uma versão assíncrona, mantenha o mesmo desenho, mas permita que handlers MCP sejam `async`:

```python
async def run_stack_sentinel_flow_async(state, llm, mcp_client):
    current = classify_intent_node(state, llm)
    route = route_by_intent(current)

    if route == "fetch_ticket":
        current = await fetch_ticket_node_async(current, mcp_client)
    elif route == "fetch_build":
        current = await fetch_build_node_async(current, mcp_client)
    elif route == "fetch_docs":
        current = await fetch_docs_node_async(current, mcp_client)
    else:
        current = fallback_node(current)

    return final_answer_node(current)
```

Essa versão é apenas um ponto de partida; em produção, adapte cada node para contrato assíncrono completo.
