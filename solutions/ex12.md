# Ex12 — Solução

Arquivo:

```text
stack_sentinel/agent/nodes.py
```

Substitua a função `fetch_ticket_node` por:

```python
def fetch_ticket_node(state: AgentState, mcp_client: MCPClient) -> AgentState:
    ticket_id = state.get("ticket_id") or extract_ticket_id(state.get("user_input", ""))
    if not ticket_id:
        return update_state(state, ticket_id=None, context=None, error="ticket_id not found")

    result = mcp_client.call_tool(TICKET_TOOL_NAME, {"ticket_id": ticket_id})
    if not result.get("ok"):
        return update_state(state, ticket_id=ticket_id, context=None, error=result.get("error", "ticket tool failed"))

    return update_state(state, ticket_id=ticket_id, context=result, error=None)
```

## Solução alternativa válida

Alguns alunos podem preferir extrair sempre o ID do texto, ignorando `state["ticket_id"]`. Isso funciona para o primeiro teste, mas falha quando o ID já vem preenchido no state. Por isso, a alternativa correta deve preservar esta ordem:

```python
ticket_id = state.get("ticket_id") or extract_ticket_id(state.get("user_input", ""))
```

O node não deve chamar a FastAPI diretamente neste exercício. Aqui o objetivo é validar que o agente usa o MCP Client, que pode ser didático ou conectado ao `FastMCP`.

## Desafios opcionais — solucao proposta

Para metadados:

```python
metadata = {"tool": TICKET_TOOL_NAME, "arguments": {"ticket_id": ticket_id}}
return update_state(state, ticket_id=ticket_id, context={"result": result, "mcp_call": metadata}, error=None)
```

Para retry simples:

```python
result = mcp_client.call_tool(TICKET_TOOL_NAME, {"ticket_id": ticket_id})
if not result.get("ok"):
    result = mcp_client.call_tool(TICKET_TOOL_NAME, {"ticket_id": ticket_id})
```

Em uma implementação real, limite retries por tipo de erro e registre a tentativa no state.
