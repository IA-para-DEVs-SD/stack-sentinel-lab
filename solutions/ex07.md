Arquivo:

```text
stack_sentinel/mcp_server/prompts.py
```

Depois de implementar o handler, exponha-o no `FastMCP` com `@mcp.prompt()`.

Substitua a função `incident_triage_prompt` por:

```python
def incident_triage_prompt(user_question: str, available_context: str) -> str:
    return (
        "Voce e um agente de triagem de incidentes. "
        f"Pergunta do usuario: {user_question}. "
        f"Contexto disponivel: {available_context}. "
        "Resuma o problema, cite severidade quando houver evidencia, "
        "sugira proximo passo e nao invente dados ausentes."
    )
```

## Registro no FastMCP

No `fastmcp_server.py`, coloque o registro dentro de uma funcao auxiliar:

```python
from stack_sentinel.mcp_server import prompts


def register_prompts(mcp):
    @mcp.prompt()
    def incident_triage_prompt(user_question: str, available_context: str) -> str:
        return prompts.incident_triage_prompt(user_question, available_context)
```

Depois chame `register_prompts(mcp)` dentro de `create_fastmcp_server()`.

## Desafios opcionais — solucao proposta

Um segundo prompt pode seguir o mesmo padrao:

```python
def build_failure_analysis_prompt(build_context: str) -> str:
    return (
        "Analise o build quebrado abaixo. "
        "Responda com Resumo, Evidencias, Hipoteses, Lacunas e Proximo passo. "
        f"Contexto: {build_context}"
    )
```

Para uma estrutura mais rigida no prompt de triagem:

```python
def incident_triage_prompt(user_question: str, available_context: str) -> str:
    return (
        "Responda usando exatamente as secoes: Resumo, Evidencias, Lacunas, Proximo passo. "
        "Nao invente dados ausentes. "
        f"Pergunta: {user_question}. Contexto: {available_context}"
    )
```
